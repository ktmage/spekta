# Spekta

**テストコードから仕様書を生成する — AI 時代のドキュメンテーションフレームワーク**

## 概要

Spekta は、テストコード（RSpec / Vitest など）を唯一の信頼できる情報源（Single Source of Truth）として、人間が読める仕様書・ドキュメントを自動生成するツールチェインです。

AI がコードを大量に生成する時代において、すべてのコードを人間がレビューするのは現実的ではありません。重要なのは**振る舞い（Behavior）**に関心を寄せること。テストコードはその振る舞いが成立しているかを明確に示す唯一の手段です。

Spekta は「テストコードと仕様書の 1:1 同期」という思想のもと、以下を実現します:

- きれいなテストを書けば、きれいなドキュメントが生成される
- ドキュメントをメンテナンスすれば、テストも同期される
- テストとドキュメントの間に乖離が生まれない

## モチベーション

- **AI 時代の品質保証**: AI が生成したコードの品質を、人間が全行レビューするのではなく、振る舞いの検証に集中する
- **テストコード駆動**: テストは振る舞いの正しさを機械的に証明できる唯一の成果物。これをドキュメントの源泉にする
- **ドキュメントの鮮度**: 手書きの仕様書はコードと乖離しがち。テストから自動生成することで、常に最新の状態を保つ

## 構成要素

### 1. テストコードの記述基準（Test Writing Convention）

テストから質の高いドキュメントを生成するための、テストコードの書き方ガイドライン。

- BDD スタイルの `describe` / `context` / `it` の構造ルール
- テストの説明文（description）の書き方規約
- メタデータ・タグの付与ルール（カテゴリ、優先度、対象機能など）
- テストの粒度と構成のベストプラクティス

### 2. テストコードの静的解析（Static Analyzer）

テストコードを AST レベルで解析し、構造化された中間表現に変換する。

- 対応フレームワーク: RSpec, Vitest（段階的に拡張）
- テストの階層構造、説明文、メタデータの抽出
- 記述基準への準拠チェック（Linter 的な役割）

### 3. ドキュメント生成（Document Generator）

中間表現から、人間が読みやすい Web ドキュメントを生成する。

- 仕様書形式の HTML / Markdown 出力
- テストの階層構造をそのままドキュメントのセクション構造に反映
- テストのパス/フェイル状態の可視化
- 検索・ナビゲーション可能な静的サイト生成

## 名前の由来

**Spekta** — 「スペクタ」はラテン語の *spectare*（観察する、見る）に由来し、英語の *spec*（仕様）と *spectra*（スペクトル、光の分解）を連想させます。テストコードという光をプリズムのように分解し、仕様書として可視化する、というコンセプトを表現しています。
