# spekta build

テストコードを解析し、Web/Markdown 形式の仕様書を生成するコマンド。

## RSpec の Feature spec から仕様書を生成する場合

### Web レンダラーの場合

#### ページごとの HTML ファイルが生成されること

1. spekta build を実行する
2. .spekta/web/ 配下にページディレクトリが存在する
3. 各ページに index.html が含まれる

#### index.html が最初のページにリダイレクトすること

1. ルートの index.html を読み込む
2. meta refresh によるリダイレクトが設定されている

#### 404.html がルートにリダイレクトすること

1. 404.html を読み込む
2. ルートへのリダイレクトが設定されている

#### HTML にページタイトルが含まれること

1. 生成された HTML を読み込む
2. タイトル要素が存在する

#### HTML にサイドバーが含まれること

1. 生成された HTML を読み込む
2. サイドバー要素が存在する

#### HTML にヘッダーのプロダクト名が含まれること

1. 生成された HTML を読み込む
2. .spekta.yml の name が表示されている

#### HTML にフッターが含まれること

1. 生成された HTML を読み込む
2. フッターに "Generated by Spekta" が含まれる

#### HTML にステップが自然言語で表示されること

1. 全ページの HTML を走査する
2. Capybara の visit が「ページを開く」に変換されている

#### HTML に see 参照がリンクとして含まれること

1. 全ページの HTML を走査する
2. [spekta:see] がリンク要素として出力されている

#### HTML に summary 属性が含まれること

1. 全ページの HTML を走査する
2. [spekta:summary] が要約要素として出力されている

#### HTML に why 属性がコールアウトとして含まれること

1. 全ページの HTML を走査する
2. [spekta:why] がコールアウト要素として出力されている

### Markdown レンダラーの場合

#### ページごとの .md ファイルが生成されること

1. .spekta/markdown/ 配下のファイルを確認する
2. .md ファイルが 1 つ以上存在する

#### Markdown にページタイトルが h1 として含まれること

1. 生成された Markdown を読み込む
2. h1 見出し（# ...）が存在する

#### Markdown にステップが番号付きリストとして含まれること

1. 全 Markdown ファイルを走査する
2. Capybara の visit が「1. ページを開く」として出力されている

#### Markdown に see 参照がリンクとして含まれること

1. 全 Markdown ファイルを走査する
2. [spekta:see] が Markdown リンクとして出力されている
